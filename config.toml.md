# config.toml — Reference

**Location**: `C:\Users\<user>\.codex\config.toml`

**Role**: Main configuration file for the Codex application. Contains the active model, providers, profiles, MCP servers, plugins, and sandbox options.

⚠️ This file is modified **automatically** by `codex-launch.ps1` on each launch via the menu.

---

## Essential Sections

### Header — active model and provider

```toml
model = "deepseek-flash"          # ← modified by the launcher (Set-Default)
model_reasoning_effort = "xhigh"  # "max" removed since codex-cli 0.118.0
personality = "pragmatic"
model_provider = "litellm"        # ← modified by the launcher (Set-Default)
sandbox_mode = "danger-full-access"
```

### Model Catalog

```toml
model_catalog_json = "C:\\Users\\<user>\\.codex\\ollama-launch-models.json"
```

This file **MUST** exist and contain the LiteLLM entries (see `ollama-launch-models.json.md`).

### Notifications

```toml
notify = [
  "<path-codex>\\codex-computer-use.exe",
  "turn-ended"
]
```

### Trusted Projects

```toml
[projects.'C:\\Serveurs\\Codex Free']
trust_level = "trusted"
```

Each project folder used with Codex must be listed here. The launcher does not modify this.

## Model Providers

### OpenAI (your account)

```toml
[profiles.openai]
model = "gpt-5-codex"
model_provider = "openai"
```

No proxy required — direct connection to OpenAI.

### Ollama local

```toml
[model_providers.ollama-launch-codex-app]
name = "Ollama"
base_url = "http://127.0.0.1:11434/v1/"
wire_api = "responses"

[profiles.local]
model = "cc-ollama"
model_provider = "ollama-launch-codex-app"
```

### Ollama cloud

```toml
[profiles.cloud]
model = "minimax-m3:cloud"
model_provider = "ollama-launch-codex-app"
```

Requires prior `ollama signin`.

### LiteLLM — bridge for DeepSeek / NVIDIA / HuggingFace

```toml
[model_providers.litellm]
name = "LiteLLM"
base_url = "http://localhost:4000/v1/"
wire_api = "responses"
env_key = "LITELLM_KEY"
model_catalog_json = "C:\\Serveurs\\Codex Free\\litellm-codex\\litellm-models.json"

```

> ⚠️ **The launcher does NOT switch via profiles.** `Set-Default` ([codex-launch.ps1](codex-launch.ps1)) rewrites the `model` / `model_provider` header (see above) directly on every launch. Valid model names — which must match the `model_list` in `litellm-codex/config.yaml` and the slugs in `litellm-models.json` — are: `deepseek-flash`, `deepseek-pro`, `nvidia-deepseek`, `nvidia-glm`, `hf`. Example written for choice 1:
>
> ```toml
> model = "deepseek-flash"
> model_provider = "litellm"
> ```

## MCP Servers

### Internal node_repl server (NEVER DELETE)

```toml
[mcp_servers.node_repl]
args = []
command = "<path-codex>\\node_repl.exe"
startup_timeout_sec = 120

[mcp_servers.node_repl.env]
NODE_REPL_NATIVE_PIPE_CONNECT_TIMEOUT_MS = "1000"
NODE_REPL_NODE_MODULE_DIRS = "<path-codex>\\node_modules"
NODE_REPL_NODE_PATH = "<path-codex>\\node.exe"
NODE_REPL_TRUSTED_CODE_PATHS = "C:\\Users\\<user>\\.codex"
CODEX_HOME = "C:\\Users\\<user>\\.codex"
BROWSER_USE_AVAILABLE_BACKENDS = "chrome,iab"
BROWSER_USE_CODEX_APP_BUILD_FLAVOR = "prod"
SKY_CUA_NATIVE_PIPE = "1"
CODEX_CLI_PATH = "<path-codex>\\codex.exe"
# + other env variables generated by Codex
```

This block is **managed by Codex** — do not modify manually.

### External MCP servers (managed by the launcher)

```toml
[mcp_servers.playwright]
args = ["@playwright/mcp@latest"]
command = "npx"

[mcp_servers.figma]
url = "https://mcp.figma.com/mcp"

[mcp_servers.supabase_<project>]
url = "https://mcp.supabase.com/mcp?project_ref=<ref>&read_only=true"

[mcp_servers.render]
url = "https://mcp.render.com/mcp"
[mcp_servers.render.headers]
Authorization = "Bearer <token>"
```

**Important**: external MCP servers stay **active for all menu choices (1→7)**. `mcp-backup.toml` is a manual backup (safety net); the launcher no longer disables them automatically. The `Strip-MCP` function still exists in `codex-launch.ps1` but is no longer triggered.

## Recovery After a Codex Update

If a Codex update overwrites `config.toml`:

1. **Restore the active model**: `model = "deepseek-flash"` + `model_provider = "litellm"`
2. **Verify `model_catalog_json`** points to `ollama-launch-models.json`
3. **Verify `[model_providers.litellm]`** is present
4. **Verify `model_reasoning_effort = "xhigh"`** (the `"max"` value is invalid since codex-cli 0.118.0)
5. **Verify `sandbox_mode = "danger-full-access"`**
6. **Verify the `[mcp_servers.*]`** external servers are present (or restore from `mcp-backup.toml`)
7. **Verify the `[projects.*]`** trusted projects
8. **Verify the `[plugins.*]`** enabled plugins

The launcher automatically handles point 1 (`Set-Default`). MCP servers (point 6) are **no longer disabled automatically** — restore them manually from `mcp-backup.toml` if a Codex update wipes them.
